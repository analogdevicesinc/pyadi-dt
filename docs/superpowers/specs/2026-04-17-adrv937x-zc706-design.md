# ADRV937x + ZC706 Support

## Summary

Add ADRV937x (AD9371) transceiver + ZC706 FPGA board support to both the
declarative System API and the XSA pipeline. The AD9371 is device-tree
identical to the ADRV9009, so the existing `ADRV9009` device class is reused.
The main additions are a ZC706 `FpgaBoard`, an ADRV937x eval board factory,
and a dedicated `ADRV937xBuilder` for the XSA pipeline.

## Motivation

The ADRV9371 + ZC706 is a supported Kuiper board combination with existing
XSA profiles but no declarative System API path and no dedicated XSA builder.
Adding these enables end-to-end device tree generation for this platform.

## Design

### ZC706 FpgaBoard

New file `adidt/fpga/zc706.py` following the ZCU102 pattern.

Key differences from ZCU102:

- `arch = "arm"` (Zynq-7000, 32-bit) vs `"arm64"`
- `addr_cells = 1` vs `2`
- 2 SPI masters, 4 GT lanes (typical ZC706 configuration)

Registered in `adidt/fpga/__init__.py` as `zc706()` factory function.

### ADRV937x Eval Board

New file `adidt/eval/adrv937x_fmc.py` following the AD9081 FMC pattern.

- Factory function `adrv937x_fmc(reference_frequency=122_880_000)`
- Wraps an `ADRV9009` device instance with `compatible = "adi,adrv9371"`
  (the device class already documents ADRV937x support)
- Pairs with an `AD9528` clock chip
- Exposes the standard eval board interface: `converter`, `clock`,
  `dev_refclk`, `sysref`, clock output ports for JESD links

Registered in `adidt/eval/__init__.py`.

### ADRV937xBuilder (XSA Pipeline)

New file `adidt/xsa/builders/adrv937x.py` — a separate builder class.

- `matches()` checks for `"ad9371"` / `"adrv937"` in topology converter
  IP types
- `build_model()` and `build_nodes()` duplicate the relevant rendering
  logic from `ADRV9009Builder` for JESD + clock + converter DT nodes.
  No shared helper extraction — keep the two builders independent to
  avoid premature coupling. A future refactor can consolidate if the
  logic stays identical.
- Registered in `adidt/xsa/builders/__init__.py` so `NodeBuilder`
  discovers it.

### Tests

- **System API unit test:** `test/devices/test_system_adrv937x_zc706.py`
  — compose ADRV937x + ZC706 via System API, verify BoardModel output
  (converter part, clock part, JESD links, SPI connections).
- **XSA builder test:** Golden DTS file
  `test/xsa/test_builders/golden_adrv937x_zc706.dts` and a test case in
  the existing builder test suite that runs ADRV937xBuilder against the
  ADRV937x ZC706 XSA profile.
- **Hardware test:** Not in scope (no ADRV937x+ZC706 hardware in the
  coordinator).

## Files

| File | Action | Responsibility |
|------|--------|----------------|
| `adidt/fpga/zc706.py` | Create | ZC706 FpgaBoard subclass |
| `adidt/fpga/__init__.py` | Modify | Export `zc706` factory |
| `adidt/eval/adrv937x_fmc.py` | Create | ADRV937x eval board factory |
| `adidt/eval/__init__.py` | Modify | Export `adrv937x_fmc` factory |
| `adidt/xsa/builders/adrv937x.py` | Create | ADRV937xBuilder |
| `adidt/xsa/builders/__init__.py` | Modify | Register ADRV937xBuilder |
| `test/devices/test_system_adrv937x_zc706.py` | Create | System API unit test |
| `test/xsa/test_builders/golden_adrv937x_zc706.dts` | Create | Golden DTS for builder test |
| `test/xsa/test_builders/` (existing test file) | Modify | Add ADRV937x builder test case |

## Out of Scope

- Extracting shared helpers between ADRV9009Builder and ADRV937xBuilder.
- Hardware integration tests for ADRV937x+ZC706.
- Changes to the existing ADRV9009 device class or builder.
